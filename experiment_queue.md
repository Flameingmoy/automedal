## 1. iso-bin-regularization [axis: preprocessing] [STATUS: done]
**Hypothesis:** Constraining isotonic regression to 50–200 bins (informed by Fisher ratio and ~1.4% estimated noise level) will improve generalization vs default sklearn isotonic because exp 0017's best result (0.051357) likely reflects isotonic overfitting on noisy validation samples — the dataset is 98.6% accurate, meaning ~1.4% of samples are mislabeled, and unconstrained isotonic regression fits these noisy points exactly, distorting the calibration mapping for clean samples; regularizing the bin count tightens the piecewise-constant mapping to the noise floor without sacrificing the ~0.0010 improvement that makes isotonic the dominant post-processing step.
**Sketch:** In `agent/train.py`, implement a custom bin-constrained isotonic calibration: sort validation samples by predicted probability per class, partition into N equal-frequency bins, assign each bin the bin-mean of true labels, then apply the piecewise-constant mapping to test predictions. Grid-search N ∈ {30, 50, 100, 200, 500, default} within 5-fold nested validation to avoid data leakage. Use the N that minimizes OOF log_loss. This is a pure post-processing step — zero Optuna budget impact. Compare against default isotonic from the same HPO run to isolate the regularization effect.
**Expected:** 0.0001–0.0003 improvement if bin regularization prevents noise overfitting; the cleanest test since it uses the same base models as exp 0017. A null result confirms isotonic's piecewise-constant mapping is already optimally constrained for this dataset's noise profile, closing this calibration sub-axis and redirecting effort to base model quality.
**Derived from:** arxiv 2512.05469 "How Ensemble Learning Balances Accuracy and Overfitting" — see research_notes.md

## 2. warm-start-optuna [axis: HPO] [STATUS: pending]

**Sketch:** In `agent/train.py`, collect all prior Optuna trial records from `agent/results.tsv` (extract XGB/LGB/CAT params and val_loss from each run's notes or a dedicated trial log). Train a simple meta-model (e.g., RandomForestRegressor or Bayesian Ridge on trial hyperparameters → val_loss) to score untried configurations. Use this meta-model as an Optuna prior: seed the next Optuna study with configurations scored as promising by the meta-model. Run full-quality Optuna HPO (same trial counts as exp 0017: ~40+ XGB, ~8+ LGB, ~30+ CAT). Apply isotonic calibration. Budget for meta-model fitting is ~5s — negligible.
**Expected:** If warm-start moves Optuna toward better basins, base model quality improves, pushing isotonic-calibrated ensemble toward 0.0510–0.0513. Even a modest improvement validates the approach; a null result confirms Optuna's random search is already near-optimal for this hyperparameter space after 20+ runs.

## 3. na-fir-calibration [axis: calibration] [STATUS: pending]
**Hypothesis:** Applying normalization-aware isotonic calibration (NA-FIR: incorporate sum-to-one constraint directly into isotonic optimization) to the XGB+LGB+CatBoost ensemble's softmax outputs will reduce log_loss below 0.051357 because standard one-vs-rest isotonic regression ignores probability normalization constraints — the per-class isotonic mappings can independently shift class probabilities in ways that violate sum-to-one, and this violation is the main source of residual error in exp 0017's isotonic plateau; enforcing normalization jointly should tighten the calibration mapping.
**Sketch:** In `agent/train.py`, implement NA-FIR calibration: for each validation sample, compute the one-vs-rest isotonic mapping per class (same as exp 0017), then post-process to enforce sum-to-one normalization: p_calibrated_c = p_iso_c / sum(p_iso_all_classes) for each class c. Alternatively, implement SCIR (cumulative bivariate isotonic regression) which models the joint distribution of two classes at a time, naturally enforcing rank constraints. Compare both against default sklearn isotonic within the same run. Use the method that minimizes OOF log_loss. This is a pure post-processing step — zero Optuna impact.
**Expected:** 0.0001–0.0003 improvement over default isotonic if sum-to-one enforcement addresses a real calibration gap. A null result confirms that the 0.0010 isotonic improvement already accounts for normalization implicitly (the models are well-calibrated individually before isotonic), closing this calibration sub-axis.
**Derived from:** arxiv 2512.09054 "Improving Multi-Class Calibration through Normalization-Aware Isotonic Techniques" — see research_notes.md

## 4. jsd-local-calibration [axis: calibration] [STATUS: pending]
**Hypothesis:** JSD-based local calibration — which weights calibration training samples by local feature-space neighborhood density rather than treating all samples equally — will improve log_loss because standard isotonic regression (exp 0017) treats every validation sample with equal importance, but miscalibration is concentrated in sparse feature regions where the ensemble has fewer neighbors to inform its predictions; weighting calibration toward high-density (well-represented) regions reduces the influence of noisy calibration signal from sparse regions.
**Sketch:** In `agent/train.py`, implement JSD-based local calibration: for each validation sample, compute the Jensen-Shannon distance to its K nearest neighbors in feature space (K=50–200, grid-search). Use 1 / (1 + JSD_dist) as sample weight for isotonic regression fitting. Compare against: (a) uniform-weight isotonic (exp 0017 baseline), (b) density-threshold isotonic (fit only on samples above median local density), (c) NA-FIR if not already tested. Grid-search K ∈ {50, 100, 200} within nested CV. Apply best method to test predictions. This is a pure post-processing step with ~10s overhead.
**Expected:** 0.0001–0.0002 improvement if JSD-weighting concentrates calibration on high-density regions. Risk: JSD computation adds overhead and the benefit may not materialize if isotonic regression's piecewise-constant mapping already handles density implicitly. A null result confirms isotonic is already near-optimal for this dataset.
**Derived from:** arxiv 2510.26566 "Multiclass Local Calibration With the Jensen-Shannon Distance" — see research_notes.md

## 5. gbdt-feature-bagging [axis: feature-eng] [STATUS: pending]
**Hypothesis:** Training each of the 3 GBDTs on a different random 60% feature subset (seeded independently) will reduce ensemble prediction correlation without adding a 4th model or halving the Optuna budget, because exp 0020 showed XGB+CAT is the most complementary pair (0.0524 vs XGB+LGB 0.0528, LGB+CAT 0.0534), implying feature-level diversity helps more than model-level diversity, and feature bagging creates exactly this diversity within the existing 3-model framework by ensuring each GBDT learns from a different feature subspace.
**Sketch:** In `agent/train.py`, before Phase 1 Optuna HPO, randomly sample 60% of features independently for each of the 3 models (XGB, LGB, CatBoost) using different random seeds. Run full Optuna HPO for each model on their respective reduced feature sets (same trial counts as the stable operating point: ~40+ XGB, ~8+ LGB, ~30+ CAT). Generate OOF predictions and grid-search ensemble weights (same 25-combo grid as exp 0017). Apply isotonic calibration. Track per-model feature subsets for reproducibility. Compare feature-bagged ensemble + isotonic vs a within-run baseline (same HPO run, full features + isotonic).
**Expected:** Individual model quality will drop modestly (60% features), but ensemble diversity should improve. If feature-bagged ensemble + isotonic beats 0.051357, this confirms decorrelation outweighs individual degradation and opens a new ensemble diversity axis. Risk: if individual degradation outweighs diversity benefit, the ensemble regresses — a null result signals that GBDTs need all available features and diversity should come from elsewhere.
**Derived from:** arxiv 2512.05469 "How Ensemble Learning Balances Accuracy and Overfitting" — see research_notes.md
