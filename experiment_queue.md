## 1. tabkd-neural-student [axis: architecture] [STATUS: done]
**Hypothesis:** Training a TabKD neural student (Tabular Knowledge Distillation with interaction-diversity synthetic queries) on the 3 GBDT teachers' softmax predictions will produce a diverse 4th ensemble member without the HPO budget penalty that killed TabR (exp 0008) and T-MLP (exp 0009), because TabKD generates synthetic training data by maximizing pairwise feature interaction coverage between the teachers — creating a fundamentally different training signal than standard KL-divergence matching — and the neural student is trained on fixed teacher predictions (no HPO budget consumed) while achieving high teacher-student agreement (14/16 configurations in the paper) on tabular data.
**Sketch:** In `agent/train.py`, after Phase 2 (3-model OOF predictions), implement TabKD training: (1) use the 3 GBDT teachers' per-sample softmax predictions to generate synthetic tabular queries that maximize coverage of pairwise feature interactions; (2) collect teacher logits on these synthetic samples; (3) train a simple 2-layer MLP student on (synthetic_features, teacher_logits) using interaction-diversity loss; (4) use a single fixed student architecture (no HPO) to avoid budget penalty; (5) add TabKD student to the ensemble at ≤10% weight (same as TabR/T-MLP ceiling). Apply 500-bin isotonic on the 3-GBDT+student ensemble. Compare 3-GBDT-only isotonic baseline (within same run) as control.
**Expected:** TabKD student individually at ~0.060–0.065 (vs TabR 0.0688, T-MLP 0.0658) if interaction-diversity training produces a better student than standard KD. If it receives ~5–10% ensemble weight, improvement of 0.0001–0.0003. Risk: neural student remains uncompetitive on this dataset regardless of training method; null result closes the neural-student axis permanently. Derived from arxiv 2603.15481 — see research_notes.md.

## 2. qxgb-uncertainty-isotonic [axis: calibration] [STATUS: pending]
**Hypothesis:** Training a fixed-config Quantile XGBoost (QXGBoost: Huber-quantile objective, 10th/50th/90th percentiles) alongside the 3 standard GBDTs and using prediction-interval width as inverse-weight for isotonic calibration will improve log_loss, because QXGBoost's per-sample uncertainty signal is absent from standard point predictions — wide intervals in sparse tabular regions indicate where the ensemble is unreliable — and weighting isotonic calibration toward high-confidence (narrow-interval) samples addresses miscalibration in exactly those regions where the JSD experiment (exp 0013) hypothesized but could not measure miscalibration concentration.
**Sketch:** In `agent/train.py`, implement QXGBoost with a single fixed config (no HPO to avoid budget penalty): `objective='reg:quantileerror'`, `quantile_alpha=[0.1, 0.5, 0.9]`, `device="cuda"`, trained on the same features as XGBoost. Extract per-sample prediction interval width = q90 - q10. In Phase 3 isotonic calibration, weight validation samples by inverse-interval-width: `weight_i = 1 / (interval_width_i + eps)`, normalized to sum to N. Compare inverse-interval-weighted isotonic vs standard unweighted isotonic (500 bins) on the 3-model ensemble. Keep 3-GBDT HPO budget unchanged. Use same base OOF predictions as exp 0010.
**Expected:** 0.0001–0.0003 improvement if uncertainty-weighted isotonic reduces influence of unreliable calibration samples. Risk: QXGBoost's uncertainty estimates are noisy on 900K-sample training, or interval width does not correlate with ensemble miscalibration — null result confirms isotonic already handles this implicitly. Null result also closes the uncertainty-calibration sub-axis.

## 3. nacp-noise-aware-calibration [axis: calibration] [STATUS: pending]
**Hypothesis:** Applying Noise-Aware Conformal Prediction (NACP) sample weights to isotonic calibration — which estimates the label noise transition matrix from validation data and down-weights samples whose labels are likely mislabeled — will improve log_loss, because the dataset's ~1.4% label noise distorts isotonic regression's piecewise-constant mapping (which treats every validation label as ground truth), and NACP's principled noise correction addresses the root cause rather than the symptom of calibration plateau at 0.051087.
**Sketch:** In `agent/train.py`, implement NACP weighting for isotonic calibration: (1) for each validation sample, compute per-class conformity scores = |pred_proba[true_class] - rank_of_this_probability_within_class|; (2) estimate per-sample noise probability by fitting a small logistic regression on (conformity_score, pred_entropy, max_proba) → label_noise_flag; (3) set isotonic sample weight = 1 - estimated_noise_prob (capped at [0.1, 1.0]); (4) apply weighted isotonic regression on the 3-model ensemble predictions with these NACP weights; (5) compare against standard unweighted isotonic (500 bins). Keep 3-GBDT HPO budget unchanged. Use same base OOF predictions as exp 0010.
**Expected:** 0.0001–0.0003 improvement if NACP's noise correction is additive to isotonic's built-in smoothing. Risk: isotonic's piecewise-constant nature already handles noisy labels implicitly by averaging over bins — NACP's correction may be redundant; null result confirms isotonic is robust to ~1.4% noise without explicit correction. Derived from arxiv 2501.12749 — see research_notes.md.

## 4. huber-loss-xgboost [axis: new-model] [STATUS: pending]
**Hypothesis:** Replacing standard multiclass logistic loss with Huber loss (transition from L2 to L1 past δ=1.0) for XGBoost will produce a more robust individual model on this ~98.6%-accurate dataset, because Huber loss's capped quadratic penalty for well-classified samples prevents overconfident gradient amplification on borderline correct predictions — the mechanism behind XGBoost's dominant 0.65 ensemble weight and the isotonic plateau — without the catastrophic focal-loss collapse (exp 0009, +0.018) that resulted from uniform down-weighting of all well-classified samples regardless of confidence level.
**Sketch:** In `agent/train.py`, implement Huber loss for XGBoost multiclass via custom objective: for each sample, compute `huber_delta=1.0`, then `gradient = (pred - label) * min(1.0, delta / |pred - label + eps|)` and `hessian = min(1.0, delta / |pred - label + eps|)`. Train XGBoost with this custom Huber objective alongside standard-log-loss LGB and CatBoost (same HPO trial counts). Grid-search ensemble weights. Apply 500-bin isotonic calibration. Compare Huber-XGB ensemble vs standard-log-loss XGB ensemble within the same run using persisted Optuna configs from the Huber XGB HPO.
**Expected:** Huber-XGB individually at ~0.0535–0.0540 (vs standard-log-loss XGB ~0.0535), with a different error profile that shifts ensemble weights away from 0.65 toward ~0.50–0.55 for XGB. If Huber-XGB maintains quality while increasing diversity, ensemble improves by 0.0001–0.0003. Risk: Huber loss is well-studied for regression but less so for multiclass; implementation bugs could degrade quality. Null result closes the noise-robust-loss sub-axis for GBDTs. Partially derived from arxiv 2310.05067 (RFL paper that inspired the loss-robustness approach).

## 5. unified-diversity-theory [axis: ensembling] [STATUS: pending]
**Hypothesis:** Applying the Unified Theory of Diversity (arxiv 2301.03962) to compute the exact optimal bias-variance-diversity tradeoff for the 3-model ensemble will reveal whether the persistent XGB-heavy weights (0.65/0.10/0.25) represent the mathematically optimal operating point for this dataset's label distribution, because the theory proves that for certain label skews, promoting diversity hurts accuracy — and if the theory predicts equal weighting (λ→0) as suboptimal (as all 15 experiments confirm), it closes the ensembling axis definitively and redirects effort toward base model quality.
**Sketch:** In `agent/train.py`, implement the diversity decomposition from arxiv 2301.03962: (1) extract per-sample binary error indicators from the 3 models' OOF predictions; (2) compute empirical bias², variance, and diversity terms for each weight configuration w ∈ {equal, 0.65/0.10/0.25, 0.55/0.10/0.35, 0.75/0.10/0.15, 0.55/0.15/0.30}; (3) compute the theoretical optimal diversity level λ* from the label-distribution-dependent formula in the paper; (4) report whether λ* predicts unequal weights as optimal; (5) if the theory predicts equal weighting, run a final experiment with equal weights + isotonic to empirically falsify the theory's prediction on this dataset. Apply 500-bin isotonic on all configurations.
**Expected:** Theory computation only (zero risk of regression). If the theory predicts unequal weights as optimal (λ* > 0), it confirms the 0.65/0.10/0.25 pattern is theoretically grounded — closing the ensembling axis and redirecting all effort to base model quality. If the theory predicts equal weighting, it challenges 15+ experiments of contrary empirical evidence and demands investigation. Derived from arxiv 2301.03962 — see research_notes.md.
