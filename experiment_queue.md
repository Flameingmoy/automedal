## 1. irredundant-kfold-hpo [axis: HPO] [STATUS: done]
**Hypothesis:** Running Optuna HPO with irredundant k-fold CV (arxiv 2507.20048, non-overlapping training partitions per fold) instead of standard overlapping k-fold will improve base model quality, because overlapping folds inflate validation correlation and produce optimistically biased variance estimates that cause Optuna to select hyperparameters overfit to the specific fold pattern — irredundant k-fold's honest variance estimates give sharper comparative distinctions between configurations, enabling Optuna to select more robust hyperparameter settings that generalize better to the held-out test set and shift the base model quality ceiling above the current 0.0534 XGB individual ceiling.

**Sketch:** In `agent/train.py`, implement irredundant k-fold CV in the Optuna objective functions: for a dataset with N samples and k folds, divide data into k equal non-overlapping blocks, each block serving as the validation set once; train on the remaining k-1 blocks (total data usage = N×k/k = N per fold, but training partitions are fully disjoint). Use the same search spaces and ~5-minute budget per model. After HPO, retrain best configs on full training set and proceed to Phase 3 (scipy.optimize SLSQP weights + 500-bin isotonic). Compare individual model quality and final val_loss vs historical best 0.050500. Derived from arxiv 2507.20048.

**Expected:** Irredundant k-fold HPO ensemble at ~0.0503–0.0507 if honest variance estimates help Optuna select more robust configs. Primary signal: whether individual XGB, LGB, and CatBoost quality improve by even 0.0001–0.0003 (which would translate to ~0.0002–0.0005 ensemble improvement via isotonic). Risk: irredundant k-fold uses non-overlapping training sets that are each (k-1)/k of data, which is the same fraction as standard k-fold, but the non-overlap may reduce effective training set diversity if the data has temporal or group structure.

## 3. early-stop-kfold-hpo [axis: HPO] [STATUS: pending]
**Hypothesis:** Implementing early stopping within Optuna's k-fold CV objective — stopping individual fold training when val_loss flattens rather than running full boosting rounds — will increase trial count by +167% within the same time budget (per arxiv 2405.03389), restoring the ~30+ XGB trial count that exp 0023 lost (only 20 XGB trials due to full-boost-round consumption of budget), and the resulting higher trial coverage will find better XGB/LGB/CatBoost hyperparameter configurations that shift the isotonic ceiling above the current 0.050500 plateau.

**Sketch:** In `agent/train.py`, implement early-stopping k-fold CV in the Optuna objective functions: for each trial, train each fold with early stopping on a held-out portion (e.g., 20% of the fold's training data as a monitor set), stopping when the monitor loss stops improving for 50 rounds. Use standard StratifiedKFold(n_splits=5) with early stopping per fold. This reduces per-trial training time from ~8s (full rounds) to ~3-4s (early stopped), enabling ~30+ XGB trials in 160s vs the 20 trials exp 0023 achieved. After HPO, proceed to Phase 3 (scipy.optimize SLSQP weights + 500-bin isotonic). Derived from arxiv 2405.03389.

**Expected:** Early-stop k-fold HPO ensemble at ~0.0503–0.0507 if the +167% trial count increase from per-fold early stopping translates to better hyperparameter configurations. Primary signal: whether XGB trial count increases from ~20 (exp 0023) to ~30+ within the same ~160s budget, and whether individual XGB quality improves. Risk: early stopping per fold may select premature stopping rounds that hurt overall model quality if the monitor set is noisy or unrepresentative.

## 4. qp-ensemble-weights [axis: ensembling] [STATUS: pending]
**Hypothesis:** Replacing scipy.optimize SLSQP gradient-based search with a constrained Quadratic Program (QP) solver for ensemble weights will find better optima, because arxiv 2512.22286 proves optimal ensemble weights are solutions to constrained QPs — SLSQP's gradient-based local search may terminate in local minima or saddle points that the QP solver avoids, and the QP formulation natively handles the sum-to-one and weight-bound constraints without penalty-function artifacts that SLSQP's augmented Lagrangian approach introduces; since the QP approach is entirely post-processing on fixed OOF predictions, it adds negligible runtime and can be stacked with any HPO improvement.

**Sketch:** In `agent/train.py`, after Phase 2 OOF predictions are generated (with clean HPO from exp 0025's approach), replace the scipy.optimize SLSQP weight optimization with a QP formulation: `min_w (w^T * C * w)` where C is the covariance/error matrix of OOF predictions, subject to `sum(w) = 1`, `w >= 0`. Solve via `scipy.optimize.minimize(method='SLSQP')` with explicit quadratic objective, or via `cvxpy` if available (`pip install cvxpy`). Compare QP val_loss vs SLSQP val_loss at the same isotonic calibration stage. QP is a post-processing step on fixed OOF predictions, adding negligible runtime. Derived from arxiv 2512.22286.

**Expected:** QP-weighted ensemble at ~0.0503–0.0507 if the QP solver finds better weight optima than SLSQP gradient search. Primary signal: whether QP weights differ meaningfully from SLSQP weights (which have converged to XGB-heavy corners across 23+ experiments). If QP and SLSQP converge to the same weights, confirms the weight optimization landscape is well-behaved and the ensembling axis closes conclusively. Risk: SLSQP already finds XGB-heavy corners robustly; QP may not improve on an already-converged optimum.

## 5. sea-accuracy-diversity [axis: ensembling] [STATUS: pending]
**Hypothesis:** Replacing scipy.optimize SLSQP fixed-weight ensembling with Self-Error Adjustment (SEA, arxiv 2508.04948) will improve val_loss because SEA's adjustable λ parameter explicitly decomposes ensemble error into self-error (individual quality) and diversity (error correlation) terms — unlike SLSQP which only minimizes log_loss directly — and the explicit λ parameter may find a better accuracy-diversity operating point than SLSQP's unconstrained optimization, especially since SLSQP's continuous convergence to XGB-heavy corners may be leaving diversity-driven gains on the table that SEA's structured decomposition can unlock; since SEA is also a post-processing step on fixed OOF predictions, it adds negligible runtime.

**Sketch:** In `agent/train.py`, after Phase 2 OOF predictions are generated, implement SEA: for each candidate λ in {0.0, 0.1, 0.3, 0.5, 0.7, 1.0, 2.0, 5.0}, compute per-model self-errors and diversity terms from OOF predictions, then compute SEA weights and evaluate isotonic-calibrated val_loss on the validation set. Select λ that minimizes val_loss. Compare SEA val_loss vs SLSQP val_loss. SEA is a post-processing step on fixed OOF predictions, adding negligible runtime (~seconds). Derived from arxiv 2508.04948.

**Expected:** SEA ensemble at ~0.0503–0.0507 if the explicit accuracy-diversity tradeoff λ parameter finds a better operating point than SLSQP's unconstrained optimization. Primary signal: whether any λ value consistently outperforms SLSQP across multiple random seeds. If SLSQP consistently outperforms SEA across all λ values, confirms SLSQP already found the optimal point and the ensembling axis closes conclusively. Risk: UTD (exp 0020) showed variance reduction dominates on this dataset, so promoting diversity via high λ may hurt.
