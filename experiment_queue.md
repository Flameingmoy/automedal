## 1. roc-ir-calibration [axis: preprocessing] [STATUS: done]
**Hypothesis:** ROC-regularized isotonic regression (ROC-IR) applied to the weighted ensemble's softmax outputs will improve log_loss because temperature scaling from exp 0002 confirmed overconfidence (temperatures ~0.94–0.95, +0.0002 signal) but the coarse grid implementation burned Optuna budget; ROC-IR is O(1) to fit and preserves multiclass ranking quality simultaneously — both calibration and ranking improve rather than trading off.
**Sketch:** In `agent/train.py`, after computing the best weighted ensemble predictions (Phase 3, no stacking), apply ROC-IR via `mapie.calibration.ROCIRCalibrator` or implement with `sklearn.isotonic.IsotonicRegression` per class using Platt scaling on the OOF predictions to regularize. Use 5-fold nested validation to prevent overfitting. Apply fitted calibrator to test predictions. This should complete in <5s and consume zero Optuna budget — preserving full trial counts for base models.
**Expected:** log_loss improvement of ~0.0001–0.0003. A null result confirms the ensemble's ranking structure is already near-optimal and calibration cannot improve it further.

## 2. covar-pseudo-label-selection [axis: pseudo-label] [STATUS: pending]
**Hypothesis:** CoVar-based pseudo-label selection (high Maximum Confidence AND low Residual-Class Variance across non-maximum classes) will produce cleaner pseudo-labels than fixed 0.95 confidence thresholds because the ensemble's 270K test predictions include high-confidence-but-wrong predictions near class boundaries that a raw max-probability threshold would include as reliable, and CoVar's variance criterion filters these out.
**Sketch:** In `agent/train.py`, generate weighted ensemble test predictions (w=0.65/0.10/0.25). Compute per-sample: (1) MC = max probability, (2) RCV = variance of probability mass across the two non-maximum classes (proxy: sum of squared non-max probs). Select rows where MC > 0.88 AND RCV < median_RCV across all test rows. Concatenate pseudo-labeled rows to training set. Retrain all 3 GBDTs (fresh Optuna HPO) on augmented dataset. Compare val_loss vs baseline (0.052421) with same budget.
**Expected:** log_loss improvement of ~0.0001–0.0004 if CoVar effectively filters mislabeled pseudo-labels near decision boundaries. Risk of degradation if RCV threshold is too aggressive and too few pseudo-labels remain.
**Derived from:** arxiv 2601.11670 "A Confidence-Variance Theory for Pseudo-Label Selection in Semi-Supervised Learning" — see research_notes.md

## 3. caruana-meta-learner [axis: ensembling] [STATUS: pending]
**Hypothesis:** The Caruana-style top-k model averaging (average predictions of the best k-model subset) as the stacking meta-learner will produce more stable results than LR stacking because LR stacking replicated unreliably across 5+ attempts (exps 0007–0010) due to multicollinearity between the three GBDT members, and the Caruana method is inherently robust to correlated base predictions by construction.
**Sketch:** In `agent/train.py`, after Phase 2 Optuna HPO for XGB/LGB/CatBoost, generate OOF predictions for each model. Compute all 7 non-empty subsets (single models, pairs, triplet). For each subset, average the OOF probabilities. Select the subset with lowest OOF log_loss. Apply that subset's averaging weights to test predictions. Compare against weighted ensemble baseline (w=0.65/0.10/0.25). This is O(1) computation — zero Optuna impact.
**Expected:** Parity or modest improvement (+/- 0.0001) over weighted ensemble. The main value is information: if any subset beats 0.052421, we know which GBDT pair/triplet is most complementary; if all subsets are worse, it confirms that model diversity is the bottleneck, not the ensembling method.
**Derived from:** arxiv 2402.01379 "Regularized boosting with an increasing coefficient magnitude stop criterion as meta-learner in HPO stacking ensemble" — see research_notes.md

## 4. tabr-fourth-member-persisted [axis: architecture] [STATUS: pending]
**Hypothesis:** TabR (feed-forward network with k-NN retrieval in middle layers) as a fourth ensemble member will reduce log_loss because DART LGB's failure (exp 0003) was structural (same error patterns, zero weight), while TabR's k-NN retrieval architecture is fundamentally different from all 3 GBDTs, providing genuine ensemble diversity that the 3-model ensemble has been unable to achieve despite 12+ experiments. Using persisted Optuna configs for the 3 GBDTs avoids the budget-halving trap.
**Sketch:** In `agent/train.py`, after Phase 1 Optuna HPO (pickle the best XGB/LGB/CatBoost configs to disk), reload them and retrain each with the saved best params (no new HPO for GBDTs). Implement TabR in PyTorch: 2-layer MLP encoder (input_dim → 256 → 128) → k-NN retrieval layer (k=32, attention-weighted) → 2-layer MLP decoder → 3-class softmax. Normalize features to [0,1]. Train TabR on GPU with batch_size=8192, lr=1e-3, 30 epochs, early stopping on val_logloss (budget ~90s). Grid-search 4-model weights with a very coarse 4-value-per-model grid (64 combos). Skip stacking to save time.
**Expected:** TabR individually is likely weaker than GBDTs (est. val_loss ~0.055–0.058), but its orthogonal architecture may contribute 5–15% ensemble weight and improve log_loss by 0.0001–0.0003. Persisting GBDT configs preserves their quality (unlike exp 0003's budget halving).
**Derived from:** arxiv 2307.14338 "TabR: Tabular Deep Learning Meets Nearest Neighbors" — see research_notes.md

## 5. jsd-local-calibration [axis: preprocessing] [STATUS: pending]
**Hypothesis:** JSD-based local calibration applied to the ensemble's probabilities will improve log_loss because standard temperature scaling treats all miscalibration equally (global scalar), but the ensemble's miscalibration is likely concentrated in sparse feature-space regions (near decision boundaries and in low-density regions), and JSD local calibration weights calibration training by local neighborhood density, addressing this proximity bias that temperature scaling cannot fix.
**Sketch:** In `agent/train.py`, after generating weighted ensemble OOF probabilities, compute for each validation sample: JSD distance to its k=64 nearest neighbors in the training feature space. Weight each sample's contribution to the calibration loss (e.g., temperature or Platt scaling) by `1 / (1 + JSD)`. Fit a per-class temperature using weighted Nelder-Mead (or L-BFGS-B). This is a post-processing step consuming <5s, preserving full Optuna budget. If JSD calibration improves, apply to test predictions.
**Expected:** log_loss improvement of ~0.0001–0.0002 in regions where the ensemble is miscalibrated. A null result confirms miscalibration is global, not local, and the ensemble is uniformly overconfident across the feature space.
**Derived from:** arxiv 2510.26566 "Multiclass Local Calibration With the Jensen-Shannon Distance" — see research_notes.md
