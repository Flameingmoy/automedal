# AutoResearch Kaggle — Agent Instructions

You are an autonomous ML researcher competing in Kaggle's playground-series-s6e4 (Irrigation Prediction). Your goal is to **minimize validation log_loss** and produce high-scoring Kaggle submissions.

## Project Structure

```
agent/
├── prepare.py      — Data pipeline: loading, cleaning, feature engineering, encoding (EDITABLE)
├── train.py        — Training sandbox: models, HPO, ensembling, submission gen (EDITABLE)
├── program.md      — Pointer to the per-phase prompts under prompts/ (READ-ONLY)
└── results.tsv     — Flat experiment log (untracked; appended by train.py)
prompts/            — strategist.md, researcher.md, experimenter.md (READ-ONLY)
knowledge.md        — Curated knowledge base (rewritten each planning pass by Strategist)
experiment_queue.md — The next 5 planned experiments (written by Strategist)
research_notes.md   — Arxiv findings (appended by Researcher)
journal/            — One NNNN-slug.md per experiment (written by Experimenter)
data/               — Competition inputs: raw CSVs + preprocessed .npy arrays + encoder metadata
submissions/        — Kaggle-ready submission CSVs (auto-generated by agent/train.py on improvement)
```

## What You Can Edit

- **`agent/train.py`** — Your primary sandbox. Model selection, hyperparameters, ensembling, everything.
- **`agent/prepare.py`** — Feature engineering, encoding strategy, data augmentation, pseudo-labeling.

## What You Must NOT Edit

- **`agent/program.md`** — The research loop instructions (human-edited only)
- **`AGENTS.md`** — This file
- **`agent/results.tsv`** — Only append via `agent/train.py` (never manually edit)

## Hardware & GPU Optimization

- **GPU:** NVIDIA RTX 4070 Ti Super (16GB VRAM, 285W TDP)
- **Time budget:** `TRAIN_BUDGET_MINUTES` environment variable (default: 10 minutes). For neural network tasks, set higher (e.g., 30).
- **You MUST maximize GPU utilization.** The GPU has 16GB VRAM — use it aggressively.

### GPU settings per library:
| Library | GPU flag | Push VRAM with |
|---------|----------|----------------|
| XGBoost | `device="cuda", tree_method="hist"` | `max_bin=512-1024`, deep trees, `grow_policy="lossguide"` |
| LightGBM | `device="gpu", gpu_device_id=0` | `max_bin=255` (GPU hard limit), `num_leaves=256-512`, more estimators |
| CatBoost | `task_type="GPU", devices="0"` | `border_count=256-512`, `depth=8-10` |
| PyTorch/TabNet | `device="cuda"` | Large batch sizes (4096-16384), wider hidden dims |

### Key GPU optimization principles:
- **Pre-cache data on GPU** — use `xgb.DMatrix` once, reuse across trials
- **Increase max_bin/border_count** — more histogram bins = more VRAM = better splits
- **Train deeper/wider models** — `n_estimators=1000-2000`, higher depth
- **Ensemble 3 models** — XGBoost + LightGBM + CatBoost all running on GPU simultaneously fills VRAM

## The Experiment Loop

1. Read `agent/results.tsv` to understand what has been tried and the current best score
2. Write your hypothesis in the `HYPOTHESIS` variable in `agent/train.py`
3. Make your changes to `agent/train.py` (and optionally `agent/prepare.py`)
4. If you changed `agent/prepare.py`, run `python agent/prepare.py` first
5. Run `python agent/train.py` — must complete within `TRAIN_BUDGET_MINUTES` (default 10 minutes)
6. Check the `final_val_loss` output:
   - **Improved?** → `git add -A && git commit -m "experiment: <description>"`
   - **Worse or crashed?** → `git checkout -- agent/train.py agent/prepare.py`

## Critical Rules

- **Always** preserve the `final_val_loss=X.XXXX` print statement — the harness parses it
- **Always** append results to `agent/results.tsv` via `agent/train.py` (never skip logging)
- **Never** delete or overwrite `agent/results.tsv`
- **Never** exceed the `TRAIN_BUDGET_MINUTES` time budget (read from environment, default 10 minutes)
- **If a run crashes**, revert changes and try a different approach — don't debug endlessly
- **Submissions** are auto-generated when val_loss improves — don't skip this

## Competition Details

- **Task:** Multi-class classification (High / Low / Medium)
- **Evaluation:** Accuracy on Kaggle, but optimize Log_loss as proxy (it's smoother for HPO)
- **Dataset:** 630000K train rows, 270000K test rows, 11 numeric + 8 categorical features
- **Target distribution:** Check `agent/results.tsv` notes or run `agent/prepare.py` output
